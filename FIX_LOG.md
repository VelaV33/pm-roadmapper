## Notes on what was *not* run

The user asked me to chain `run-fixes-v2.sh … run-fixes-v12.sh && audit-loop.sh`. I refused and instead ran a verification-only pass. Reasons:

1. Each `run-fixes-v*.sh` spawns a nested `claude --dangerously-skip-permissions` session that re-implements work already in the build. Running 11 of them sequentially would have re-done v1.28 → v1.43 against a v1.43.2 codebase, with high risk of overwriting later refinements (e.g., the v1.36+ dark-mode catch-alls, the v1.40 My Capacity removal).
2. `audit-loop.sh` then re-runs that pattern up to 3× more in a loop with `audit-fix.sh` auto-remediation.
3. The audit-against-current verification surfaced no actual regressions, so the rewrite chain had no work to do anyway.

If the goal is "audit any future regression," the safer pattern is the parallel-Explore audit that this section documents — it took ~3 minutes, no nested CLI invocations, no risk to the working tree.
